# 🚀 Полная настройка локальной среды (Background Agent → Local)

Это руководство поможет настроить локальную среду разработки для **полного соответствия** background agent окружению.

## 📋 Требования

- **Node.js**: `v22.16.0` (точная версия!)
- **npm**: `v10.9.2` 
- **Git**: для клонирования репозитория
- **nvm**: для управления версиями Node.js

## 🔧 Пошаговая настройка

### 1. 📥 Установка nvm (если не установлен)

**macOS/Linux:**
```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc
```

**Windows:**
```bash
# Скачайте и установите nvm-windows:
# https://github.com/coreybutler/nvm-windows/releases
```

### 2. 🔧 Установка точной версии Node.js

```bash
# Установить Node.js v22.16.0
nvm install 22.16.0

# Использовать эту версию
nvm use 22.16.0

# Сделать версию по умолчанию
nvm alias default 22.16.0

# Проверить версию
node --version  # должно быть v22.16.0
npm --version   # должно быть 10.9.2
```

### 3. 📂 Клонирование и настройка проекта

```bash
# Клонировать репозиторий
git clone <your-repo-url>
cd finsights-ai

# Переключиться на правильную версию Node.js (если есть .nvmrc)
nvm use

# Проверить версии
npm run check-env
```

### 4. 🏗️ Автоматическая настройка

```bash
# Запустить скрипт автоматической настройки
npm run setup-local
```

Этот скрипт проверит:
- ✅ Версии Node.js и npm
- ✅ Наличие конфигурационных файлов
- ✅ Установку зависимостей
- ✅ Структуру проекта

### 5. 📦 Установка зависимостей

```bash
# Установить точные версии зависимостей
npm install --legacy-peer-deps

# При проблемах используйте:
npm install --force
```

### 6. 🚀 Запуск проекта

```bash
# Запустить в режиме разработки
npm start

# Проект будет доступен на http://localhost:3000
# Браузер НЕ откроется автоматически (как в background agent)
```

## 📁 Структура конфигурационных файлов

После настройки у вас должны быть следующие файлы:

```
finsights-ai/
├── .nvmrc                    # Фиксация версии Node.js
├── .env.local               # Переменные окружения
├── package-lock-exact.json  # Точные версии зависимостей
├── setup-local.js           # Скрипт автоматической настройки
├── LOCAL_SETUP_GUIDE.md     # Это руководство
├── kaspi_pdf_fixes_report.md # Документация исправлений
└── ... (остальные файлы проекта)
```

## 🔧 Ключевые настройки окружения

### `.env.local` - Переменные окружения:
```bash
BROWSER=none                 # Не открывать браузер автоматически
PORT=3000                   # Порт разработки
HOST=localhost              # Хост
SKIP_PREFLIGHT_CHECK=true   # Пропуск проверок React Scripts
FAST_REFRESH=true           # Быстрое обновление
NODE_OPTIONS=--max_old_space_size=4096  # Увеличение памяти
```

### `.nvmrc` - Версия Node.js:
```
v22.16.0
```

## 🔍 Проверка корректности настройки

### 1. Проверка версий:
```bash
node --version    # v22.16.0

```

### 2. Проверка приложения:
- Откройте http://localhost:3000
- Проверьте консоль браузера (F12)
- Загрузите PDF выписку для тестирования парсера

### 3. Проверка PDF.js:
- В консоли браузера должен быть доступен `window.pdfjsLib`
- При загрузке PDF должны появляться подробные логи

## 🐛 Устранение проблем

### Проблема: Неправильная версия Node.js
```bash
nvm use 22.16.0
# или
nvm install 22.16.0 && nvm use 22.16.0
```

### Проблема: Конфликты зависимостей
```bash
rm -rf node_modules package-lock.json
npm install --legacy-peer-deps
```

### Проблема: PDF.js не загружается
- Проверьте подключение в `index.html`
- Откройте Network tab в DevTools
- Убедитесь, что CDN доступен

### Проблема: Приложение не запускается
```bash
# Очистка кеша npm
npm cache clean --force

# Переустановка
rm -rf node_modules
npm install --force

# Проверка портов
lsof -ti:3000 | xargs kill -9  # Убить процессы на порту 3000
```
